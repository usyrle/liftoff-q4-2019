Making a Capstone

  • For 8 weeks, you won't want something too big
  • Build on what you know: CRUD
  • Work on something you're passionate about!
  • MVP, or Minimum Viable Product

Capstone Requirements

  1. Demonstrate Marketable Skills
  2. Demonstrate Ability to Learn New Skills
  3. Solve a Problem
  4. Build an App Yourself*
  5. Include 3-5 Significant Features

How to Manage a Project

  • First there was waterfall...
  • ...and now, there is agile
  • ...and they look nothing alike

"The Agile Manifesto": we can't plan for everything, so let's make good, working software instead.

Guess What?

  • You're bad at estimating time. (So am I.)
  • You're bad at focusing on long, large tasks. (So am I.)
  • You're bad at patiently waiting to see results. (So am I.)

To mitigate, we want to be agile: start small, build small. Agile isn’t how fast you can go, it’s how fast you can turn.

Story / Feature Breakdown

As a ___ I want ___ So that ___

  • Features are written as epics and distilled into user stories
  • Epics are your 30,000 ft view; stories are how you get there
  • Epics & stories kept in your "backlog"
  • Is your feature "significant"?
