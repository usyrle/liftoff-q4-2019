Live Coding: What?

  • Also "whiteboarding"
  • Solve a problem on the spot
  • Often in person, but can be virtual - ask!
  • Likelihood: fairly likely

Live Coding: Why?

Interviewers want to see:

  • How do you collaborate?
  • How do you handle pressure?
  • How do you deal with roadblocks? With mistakes?
  • Above all: how do you problem solve?

Live Coding: How?

  • Gather Requirements
  • Break the Problem Down
  • Pseudocode
  • Write Syntax
  • Test
  • Make Changes

Gather Requirements

  • Ask questions and clarify
  • Helpful to write requirements

Break the Problem Down

  • Rough order of problems
  • Don't code yet - plan!

Pseudocode

  • Basically loops, if/then/else logic

Write Syntax

  • Optional...ish
  • Talk through your process out loud
  • Work through mistakes!

Test

  • Step through code
  • Test edge cases

Make Changes

  • Discuss and fix bugs
  • Discuss potential improvements!

Example: Questions you might find

  • Easy: How would you check if a string is a palindrome?
  • Medium: How would you implement a bubble sort algorithm?
  • (Very) Hard: How would you design a vending machine?

Activity: Peer Live Coding

Write a program that outputs the string representation of numbers from 1 to n. But:

  • For multiples of three, print "Fizz" instead.
  • For multiples of five, print "Buzz" instead.
  • For numbers that are both multiples of three and of five, print "FizzBuzz" instead.
