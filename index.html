"The Agile Manifesto": we can't plan for everything, so let's make good, working software with what we know.

Hard truths of programming

  • You're bad at estimating time. (So am I.)
  • You're bad at focusing on long, large tasks. (So am I.)
  • You're bad at patiently waiting to see results. (So am I.)

...so what do we do?

Why agile?

  • To mitigate all that bad stuff, we want to start small, make small changes, and build piece-by-piece.
  • Multiple flavors of agile to achieve this: Scrum and Kanban most common
  • It's also a serious buzzword in tech!

    • Agile isn’t how fast you can go, it’s how fast you can turn.

    Features and user stories

    As a ___ I want ___ So that ___

    • Features are written as epics and distilled into user stories
    • Epics are your 30,000 ft view; stories are how you get there
    • Epics & stories kept in your "backlog"
    • Is your feature "significant"?

    Let's talk Minimum Viable Product (MVP)

    • It doesn't have to be pretty: focus on usable + useful
    • Cheaper to build a few important features then test the market!
    • Meant to be iterated on - if users buy it, you need to keep selling!
    • Needs to be scoped similar to user stories: "right-sized"

    User story ideation

    • A car MVP: what about the Model T?
    • A drying machine MVP: it's just a clothesline
