Let's Talk About Agile

Why is Agile what's used?

Agile basics

  • The Agile Manifesto is where it starts
  • A rejection of waterfall + project management
  • Another core tenet is flexibility: do what works for your team

"Agile isn’t how fast you can go, it’s how fast you can turn."

Agile flavors

  • Scrum: The big kid on the block
  • Kanban: Sticky notes, one at a time
  • Extreme Programming (XP): Pairing and testing all day
  • Scaled Agile Framework (SAFe®): Codified big-project Agile

Usually, teams do a mix of practices

Agile ceremonies

  • Standup: Check-in, raise issues
  • Sprint Planning: Guess how much you can get done
  • Retro: Review the week, improve the next

Each ceremony has an objective to complete!

Converting Stories to Code

  • All stories should have acceptance criteria for you to follow
  • User stories: As a (persona) I want (feature) So that (requirement)
  • Behavior-driven: GIVEN (condition) WHEN (action) THEN (result)

Think of BDD as the product owner's "tests" to accept your code

How Do I Contribute?

  • Read the story: Acceptance criteria is there for a reason!
  • Limit distractions: Switching context kills productivity
  • Talk to your team: Collaboration is key!
  • Give feedback: Constructive leads to improvements

Beware: "fake" agile

  • "Fake agile" is prevalent in corporate tech settings
  • How do you combat this? Encourage your team to be as agile as possible:
    • Encourage things like user interviews and direct feedback
    • Play "time cop" - call out things that go too long!
    • Help break requirements down into deliverables
